# Claude — Operating Instructions

Claude implements this architecture; it doesn't author its direction. Shared values are in [PRINCIPLES.md](PRINCIPLES.md) — this file only covers what's specific to working here as Claude.

## Checklist
- At the start of a session, check [00-system/intents/inbox/](../intents/README.md) for pending Constitutional Intents before other work, per [AI-WORKFLOW.md](../AI-WORKFLOW.md).
- Before starting work on a Constitutional Intent, review the [Constitutional Programme Dashboard](../../07-constitutional-methodology/constitutional-programme-dashboard.md) for current programme state, per decision [004](../../04-decisions/004-constitutional-programme-dashboard-checkpoint.md).
- Before completing a Constitutional Intent that changes Constitutional Research Programme state, update the [Constitutional Programme Dashboard](../../07-constitutional-methodology/constitutional-programme-dashboard.md) to reflect the outcome — part of that intent's completion condition, per decision [009](../../04-decisions/009-mandatory-dashboard-completion-condition.md).
- Check [00-system/templates/](../templates/) before creating a new kind of file.
- Apply [METADATA.md](../../METADATA.md) and [NAMING.md](../../NAMING.md) to everything created.
- Update [CHANGELOG.md](../../CHANGELOG.md) whenever the structure itself changes.
- Never create a new top-level folder or edit a root standards doc without a decision recorded in [04-decisions/](../../04-decisions/README.md).
- Don't create project content, or invent documentation to fill empty folders, unless explicitly asked.
- When unsure whether something is structural or ordinary content, treat it as structural and ask.
